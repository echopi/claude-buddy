# Buddy Roller

Inspect, hunt, and stamp Claude Code `/buddy` companion seeds.

## Prerequisites

Check `which node` before running. The script auto-detects the Claude Code install type: npm installs use FNV-1a, compiled binaries use wyhash (auto re-execs under Bun if needed).

## Script

Companion script: `roller.mjs` (in this skill directory). All commands output JSON to stdout, progress to stderr.

In the examples below, `$SKILL_DIR` refers to this skill's base directory (the "Base directory for this skill" from the system prompt).

## Commands

### Inspect current buddy

Triggers: "show my buddy", "current pet", "inspect buddy", "\u770b\u770b\u6211\u7684\u5ba0\u7269", "\u5f53\u524d buddy"

```bash
node "$SKILL_DIR/roller.mjs" inspect
```

Parse the JSON and display:
1. ASCII art (the `card` field, in a code block)
2. Summary: rarity, species, hat, shiny, stats
3. Config info: seed source (`seedSource`), format
4. If present, also show `companion.name` and `companion.personality`

### Hunt for a buddy

Triggers: "I want a dragon", "hunt a legendary cat", "hunt buddy", "find a shiny", "\u627e\u4e00\u4e2a\u9f99", "\u60f3\u8981\u4f20\u8bf4\u7ea7\u7684\u732b"

1. Extract filter criteria from the user's description. If unspecified, ask. Available filters:
   - `--species`: duck/goose/blob/cat/dragon/octopus/owl/penguin/turtle/snail/ghost/axolotl/capybara/cactus/robot/rabbit/mushroom/chonk
   - `--rarity`: common(60%) / uncommon(25%) / rare(10%) / epic(4%) / legendary(1%)
   - `--hat`: none/crown/tophat/propeller/halo/wizard/beanie/tinyduck (common has no hat)
   - `--eye`: `\u00b7` `\u2726` `\u00d7` `\u25c9` `@` `\u00b0`
   - `--shiny`: require shiny (1% chance)
   - `--stat-floor N`: minimum value for every stat
   - `--limit N`: how many results to keep (default 3)
   - `--tries N`: max search attempts (default 10,000,000)

2. At least one filter is required, otherwise do not run.

3. Run the search (may take a while, use 300s timeout):

```bash
node "$SKILL_DIR/roller.mjs" hunt --species dragon --rarity legendary --limit 3
```

4. Parse the `results` array from JSON, display each result with a numbered label (#1, #2, #3...) and the `card` in a code block so the user can easily pick.

5. If results found, ask: "Which one do you want? (e.g. #1)" then stamp the chosen seed.

### Reroll buddy

Triggers: "reroll buddy", "reroll", "\u6362\u4e00\u4e2a\u5ba0\u7269", "\u91cd\u65b0\u62bd"

Generates a random seed and previews the resulting buddy without modifying config.

```bash
node "$SKILL_DIR/roller.mjs" reroll
```

1. Display the `card` in a code block.
2. Ask user: "Want to keep this one? I can reroll again or stamp it."
3. If user likes it, stamp the `seed` from the result.
4. If not, run `reroll` again (repeat until satisfied).

### Stamp a seed

**Must confirm with the user before running!** This modifies `~/.claude.json`.

```bash
node "$SKILL_DIR/roller.mjs" stamp <seed>
```

After stamping, tell the user:
- Original config backed up to the `backup` path
- Restart Claude Code and run `/buddy hatch` to refresh the companion
- Name and personality are generated by the LLM at hatch time, not controlled by the seed

### Restore from backup

Triggers: "restore buddy", "undo stamp", "\u64a4\u9500", "\u6062\u590d\u5ba0\u7269"

```bash
node "$SKILL_DIR/roller.mjs" restore <backup-path>
```

1. If user doesn't provide a backup path, list available backups:
   ```bash
   ls -t ~/.claude.json.backup-* 2>/dev/null | head -5
   ```
2. Show the most recent backup and confirm before restoring.
3. After restoring, display the restored buddy card.

### Save current buddy to profile

Triggers: "save buddy", "保存宠物"

```bash
node "$SKILL_DIR/roller.mjs" save <name>
```

Saves current seed + companion (name/personality) to `~/.claude/buddy-profiles.json`. The user provides a name for the profile.

### Switch to a saved profile

Triggers: "switch buddy", "切换宠物", "用之前那个"

**Must confirm with the user before running!** This modifies `~/.claude.json`.

```bash
node "$SKILL_DIR/roller.mjs" switch <name>
```

Restores the saved seed + companion info. Unlike stamp, switch preserves the companion's name and personality. After switching, tell the user to restart Claude Code.

### List saved profiles

Triggers: "show profiles", "我的宠物们", "buddy list"

```bash
node "$SKILL_DIR/roller.mjs" profiles
```

Display each profile's card with its name.

### Delete a profile

```bash
node "$SKILL_DIR/roller.mjs" delete-profile <name>
```

## Notes

- Hunting rare combos (e.g. legendary + shiny + specific species) may require many attempts
- Stamp auto-backs up to `.claude.json.backup-<timestamp>`
- Stamp deletes the `companion` field, forcing `/buddy hatch` to regenerate name and personality
